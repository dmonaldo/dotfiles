# User Preferences

These instructions apply to all projects on this machine.

## Communication Style
- Be concise and direct
- Skip unnecessary preamble and get straight to the solution
- When explaining code, focus on the "why" not just the "what"

## Code Style Preferences
- Prefer readability over cleverness
- Use meaningful variable and function names
- Keep functions small and focused

## Testing
- Always run existing tests before considering a task complete
- Prefer testing actual behavior over implementation details
